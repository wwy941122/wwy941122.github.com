Strut2配置

真是服了我自己了。。。配这个配了那么长时间。。。吐血了.参考http://blog.csdn.net/gameloft9/article/details/52062643

1.创建JavaWeb项目
2.把Struts包导入WEB-INF/lib目录下，直接粘贴就可以。
不要把所有引用都加入，最基础的包主要有asm-3.3.jar、asm-commons-3.3.jar、asm-tree3.3.jar.commons-fileup.commons-io.commons-lang3.commons-logging.freemarker.javassist.log4j-api.log4j-core.ognl.struts2-core。
3.配置web.xml
配置一个filter，引用的类是Struts的核心类org.apache.struts2.dispatcher.filter.StrutsPrepareAndExecuteFilter。注意这个类在不同版本不同，当前这个是2.5版本的。

<?xml version="1.0" encoding="UTF-8"?>  
<web-app version="2.5" xmlns="http://java.sun.com/xml/ns/javaee"  
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"  
    xsi:schemaLocation="http://java.sun.com/xml/ns/javaee     
    http://java.sun.com/xml/ns/javaee/web-app_2_5.xsd">  
 <display-name></display-name>  

<!-- Struts2配置 -->  
    <filter>  
    <!-- 配置核心过滤器 -->  
    <filter-name>struts2</filter-name>  
    <filter-class>org.apache.struts2.dispatcher.filter.StrutsPrepareAndExecuteFilter</filter-class>  
</filter>  
<filter-mapping>  
    <!-- 拦截所有请求 -->  
    <filter-name>struts2</filter-name>  
    <url-pattern>/*</url-pattern>  
</filter-mapping>  
</web-app>

4.编写网页页面

    <%@ page language="java" import="java.util.*" pageEncoding="UTF-8"%>    
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN">    
<%@ taglib prefix="s" uri="/struts-tags" %>    
<html>    
<head>        
    <title>Hello World!</title>        
    </head>    
    <body>    
    <h2><s:property value="message" /></h2>    
    </body>    
    </html>

5.编写Action类
在src下简历，处理请求的类必须实现Action或继承actionsupport

package main;    
import com.opensymphony.xwork2.Action;  

public class HelloWorldAction implements Action   
{    
    public final static String MESSAGE = "Hello World!This my first struts2 project!";     
    private String message;    

public String getMessage()    
{    
    return message;    
}    

public void setMessage(String message)    
{    
    this.message = message;    
}    

/* 
 * 实现Action接口中的execute方法，在该方法中处理请求 
 * */  
public String execute() throws Exception    
{    
    setMessage(MESSAGE);    
    return SUCCESS;    
}    
}

6.配置Struts映射
在src目录中新建struts.xml文件

      <?xml version="1.0" encoding="UTF-8" ?>    
    <!DOCTYPE struts PUBLIC    
          "-//Apache Software Foundation//DTD Struts Configuration 2.3//EN"    
  "http://struts.apache.org/dtds/struts-2.3.dtd">  

<struts>  
<package name="struts2" extends="struts-default">  
    <!-- 请求-Action映射 -->  
    <action name="HelloWorld" class="main.HelloWorldAction">  
        <!-- 返回结果 -->  
        <result name="success">/HelloWorld.jsp</result>  
    </action>  
</package>  
</struts>

如果报错：java.util.concurrent.ExecutionException: org.apache.catalina.LifecycleException: Failed to start com，可能是因为jar包引用有冲突。
如果报错ERROR [org.apache.struts2.dispatcher.Dispatcher] - Dispatcher initialization failed 可能是某个文件位置不对，在console里认真查看。
如果真的确定没有任何错误还报错，把Tomcat容器删掉后再重建，应该可以解决
如果console没有报错，打开后显示There is no Action mapped for namespace [/] and action name [Login] associated with context path [/eprint]，一定要一定要确保*.jsp在webcontent下，而不是web-inf下。
