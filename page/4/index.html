opencv for Android

为了做创新项目，研究了一段时间opencv4Android，做了一个小demo。实现读取手机上的图片文件，然后做成灰度图.

public class MainActivity extends Activity {
    private Button button;
    private ImageView imageView;
    private int SELECT_FILE = 1;
    private String TAG = "Tag";

    private String selectPath = null;

    private Bitmap srcBitmap;
    private Bitmap greyBitmap;
@Override
protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    button = (Button) findViewById(R.id.button);
    imageView = (ImageView)findViewById(R.id.imageView);

    button.setOnClickListener(new View.OnClickListener() {
        @Override
        public void onClick(View v) {
            selectPic();
        }
    });
}

@Override
protected void onResume() {
    super.onResume();
    OpenCVLoader.initAsync(OpenCVLoader.OPENCV_VERSION_2_4_10, getApplicationContext(), mLoaderCallback);
    //System.out.println("onResume sucess load OpenCV...");
    Log.i(TAG,"onResume sucess load OpenCV...");

}

private BaseLoaderCallback mLoaderCallback = new BaseLoaderCallback(this) {
    @Override
    public void onManagerConnected(int status) {

        switch (status){
            case BaseLoaderCallback.SUCCESS:
                //System.out.println("成功加载");
                Log.i(TAG, "成功加载");
                break;
            default:
                super.onManagerConnected(status);
                //System.out.println("加载失败加载失败");
                Log.i(TAG,"加载失败");
        }
    }
};
public void procSrc2Gray(){
    Mat rgbMat = new Mat();
    Mat greyMat = new Mat();
    BitmapFactory.Options options = new BitmapFactory.Options();
    srcBitmap = BitmapFactory.decodeFile(selectPath,options);
    greyBitmap = Bitmap.createBitmap(srcBitmap.getWidth(), srcBitmap.getHeight(), Bitmap.Config.RGB_565);
    Utils.bitmapToMat(srcBitmap, rgbMat);
    Imgproc.cvtColor(rgbMat, greyMat, Imgproc.COLOR_RGB2GRAY);
    Utils.matToBitmap(greyMat, greyBitmap);
    System.out.println("procSrc2Gray sucess");
    imageView.setImageBitmap(greyBitmap);
    //Log.i(TAG,"procSrc2Gray sucess");

}



private void selectPic(){
    Intent intent1 = new Intent(Intent.ACTION_PICK, MediaStore.Images.Media.EXTERNAL_CONTENT_URI);
    intent1.setType("image/*");
    startActivityForResult(Intent.createChooser(intent1, "选择开启图库"),SELECT_FILE);

}


@Override
protected void onActivityResult(int requestCode, int resultCode, Intent data) {
    super.onActivityResult(requestCode, resultCode, data);
    if(resultCode == Activity.RESULT_OK){
        if(requestCode == SELECT_FILE)
            onSeclectFromGalleryResult(data);
    }

}
private void onSeclectFromGalleryResult(Intent data){
    Uri selectedImageUri = data.getData();
    String[] projection = {MediaStore.MediaColumns.DATA};
    Cursor cursor = managedQuery(selectedImageUri,projection,null,null,null);
    int column_index = cursor.getColumnIndexOrThrow(MediaStore.MediaColumns.DATA);
    cursor.moveToFirst();

    selectPath = cursor.getString(column_index);
    procSrc2Gray();


}

Android笔记-Service

service的一些基本操作

service是系统四大组件之一，其作用是在后台静默的执行一些操作而不被使用者感知。

1.service的组成
onCreate(), onStartCommand, onDestroy. 其中，onCreate只在第一次创建时才会调用。
同时需要在manifest文件注册service。

2.如何启动一个service
与启动Activity类似，service的启动也是用intent，调用方法startservice(intent)即可。
也可以用putExtra()传递数据。

2.在service中实现功能
需要在service的onCreate或者onStartCommand()方法里创建一个Thread

4 .绑定service，实现活动和服务之间的通信
把Activity与Service绑定在一起，那就是Service要随着程序的退出而终止。在Activity中根据具体的场景来调用MyBinder中的任何public方法，即实现了Activity指挥Service干什么Service就去干什么的功能（调用class MyBinder extends Binder）

首先创建了一个ServiceConnection的匿名类，在里面重写了onServiceConnected()方法和onServiceDisconnected()方法，这两个方法分别会在Activity与Service建立关联和解除关联的时候调用。在onServiceConnected()方法中，我们又通过向下转型得到了MyBinder的实例，有了这个实例，Activity和Service之间的关系就变得非常紧密了
　　　　
首先通过使用Activity的bindService(Intent service,ServiceConnection conn,int flags)

bindService(new Intent(this,MyService.class),this,     Context.BIND_AUTO_CREATE);

然后编写类实现ServiceConnection 接口

   public void onServiceConnected(ComponentName name, IBinder service) {
    //System.out.println("Service connected");

}

@Override
public void onServiceDisconnected(ComponentName name) {
    //System.out.println("ServiceDisconnected");
}

同时重写service的onBind(Intent intent)方法

@Override
public IBinder onBind(Intent intent) {
    return new Binder();
}

5.IntentService
一般的服务启动后会一直运行，除非有stopSelf方法，和stopService方法才会停止，IntentService却可以自动结束。 但只需调用onHandleIntent() 方法即可实现服务自动结束。

6.service 和 Thread 的关系
没有任何关系。
service 用于Activity被销毁后，在后台继续执行动作。也就是说，如果在服务里执行耗时长的动作，还是会ANR。 而子线程就用于避免这种状况。

Android笔记-application

Application是Android框架的一个系统组件

Application对象的生命周期是整个程序中最长的，它的生命周期就等于这个程序的生命周期。通过Application来进行一些，如：数据传递、数据共享和数据缓存等操作。是一个真正的“全局变量”
通常我们是不需要指定一个Application的，系统会自动帮我们创建。
如何自己创建一个Application 类呢？
1.创建一个class

import android.app.Application;
import android.content.res.Configuration;


public class App extends Application {
    private String appdata = "default";            //定义一个全局的变量

    public String getAppdata() {
        return appdata;
    }

    public void setAppdata(String appdata) {
        this.appdata = appdata;
    }

2.在AndroidManifest文件里注册

<application
        android:name="CustomApplication">
</application>

Android笔记-Intent

Intent的一些基本用法

Intent作用之一是用于Activity之间的来回切换（），是一个Activity启动另外一个必须要使用的组件。分为显式和隐式。
1.显式Intent
使用例子如下：

Button btn1 = (Button)findViewById(R.id.btn1);
btn1.setOnClickListener(new View.OnClickListener() {
    @Override
    public void onClick(View v) {
        Intent intent = new Intent(MainActivity.this, Activity2.class);
        startActivity(intent);
    }
});

2.隐式Intent
使用例子如下：

    findViewById(R.id.btn2).setOnClickListener(new View.OnClickListener() {
        @Override
        public void onClick(View v) {
            startActivity(new Intent("io.github.wwy941122.intenttest.intent.ACTION"));
        }
    });
}

此时只需要知道Activity在Manifests中的名称就可以启动。

<intent-filter>
    <action android:name="android.intent.action.MAIN" />   //此处的名称

    <category android:name="android.intent.category.LAUNCHER" />
</intent-filter>

Intent在两个Activity过程中还可以传递数据，用到putextra()、getextra()这两个个方法，可传递String和Bundle等

Android 笔记：activity

最近开始学Android，用的书是《第一行代码》，同时看极客学院的视频。

Android最基础的部分是activity，就是活动。Android上一切都建立在activity上。

创建活动

在as上创建很简单，只用选blank activity就把所有工作完成了。实际上创建一个activity要完成的工作有

  • 新建类。继承（extends）activity基本类
  • 创建和加载布局
  • 在Manifest文件里注册

Activity的生命周期

生命周期分为onCreate、onStart、onResume、onPause、onRestart、onStop、onDestroy。除onDestroy外前面的两两……………………………………………………………….配对。

启动模式

启动模式分为Standard（默认）、SingleTask、SingleTop、SingleInstanse.

1.Standard

standard模式就是无论什么情况下，一旦激活activity，就会在任务栈里新建一个实例，退出就会销毁。

2.SingleTop

这个模式是，如果激活自己，就不会创建。即当前activity若是在栈顶，就不会新建实例。其他情况还是会创建新的。

3.SingleTask

这个模式下的一个activity只会有一个实例，每次激活的都是这个实例。
只需要把此Activity放入栈顶，并把该Activity以上的Activity实例都pop这条待验证

4.SingleInstance

这个模式下激活会启用一个新的栈结构存放activity，与原结构完全独立。
