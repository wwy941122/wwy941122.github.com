Android笔记-网络

1.Http协议
http（超文本传输协议）是一个基于请求与响应模式的、无状态的、应用层的协议，常基于TCP的连接方式

2 使用HttpClient

HttpClient是Apache提供的http网络访问接口。但在6.0里被禁用了，5.0也不被推荐使用了。
以下是一个demo：

class NetworkThread extends Thread{
public void run(){
    HttpClient httpClient = new DefaultHttpClient();
    HttpGet httpGet = new HttpGet("http://www.baidu.com");
    try {
        HttpResponse httpResponse = httpClient.execute(httpGet);

        int code = httpResponse.getStatusLine().getStatusCode();

        if(code == 200) {
            HttpEntity entity = httpResponse.getEntity();
            InputStream inputStream = entity.getContent();
            BufferedReader bf = new BufferedReader(new InputStreamReader(inputStream));
            String line = bf.readLine();
            Log.d("HTTP","服务器"+line);
            Message msg = new Message();
            msg.obj = line;
            handler.sendMessage(msg);
        }


    } catch (IOException e) {
        e.printStackTrace();
    }
}

3.使用HttpURLConnetion（推荐）
调用HttpURLConnection类的openConnect方法，然后设置各种性质即可。比如设置请求随时用的方法、设置连接超时等等。服务器返回的是输入流，需要用BufferReader读取。

     private void sendRequestWithHttpURLConnection(){
    new Thread(new Runnable() {
        @Override
        public void run() {
            HttpURLConnection connection = null;
            try {
                URL url = new URL("http://www.baidu.com");
                connection = (HttpURLConnection) url.openConnection();
                connection.setRequestMethod("GET");
                connection.setConnectTimeout(8000);
                connection.setReadTimeout(8000);

                InputStream in = connection.getInputStream();
                BufferedReader reader = new BufferedReader(new InputStreamReader(in));
                StringBuilder response = new StringBuilder();
                String line;
                while ((line = reader.readLine()) != null){
                    response.append(line);
                }

                Message msg = new Message();
                msg.what = SHOW_RESPONSE;
                msg.obj = response.toString();
                handler.sendMessage(msg);
            } catch (MalformedURLException e) {
                e.printStackTrace();
            } catch (IOException e) {
                e.printStackTrace();
            }finally {
                if (connection != null){
                    connection.disconnect();
                }
            }

        }
    }).start();


}

Android笔记-异步消息处理机制

Android的异步消息处理机制，即handler、looper和messagequeue三者的关系。

1.线程

关于线程的两种实现方式、生命周期等等，这些java中的知识就不赘述了。
Android中的线程分为MainThread（主线程）和WorkerThread（子线程）。只有MainThread可以刷新UI（除progressbar.setProgress），用WorkerThread进行UI操作可能导致程序崩溃。

2.异步消息处理

以上的问题的解决办法，具体来说就是：建立一个子类继承handler，然后在主线创建handler实例。然后在子线程创建Message实例（handler.obtainMessage()），用handler.sendmessage()把message送出到Messagequeue，最后Looper把Message取出，再用handler对象中的handleMessage方法处理消息。
图图图

3.实现原理

  1. 实现这个机制主要利用了java的ThreadLocal
    static final ThreadLocal<Looper> sThreadLocal = new ThreadLocal<Looper>();
    sThreadLocal.set(new Looper(quitAllowed));
    。使一个线程只有一个looper对象与之对应
    ThreadLocal的set()把线程对象作为键，生成键值对。

  2. Handler构造方法中 mLooper = Looper.myLooper();。这样，Looper、Thread、Handler就对应起来。一个Handler，一个loop，一个messagequeue。

  3. Loop.loop()
    public static void loop() {

    final Looper me = myLooper();
if (me == null) {
    throw new RuntimeException("No Looper; Looper.prepare() wasn't called on this thread.");
}
final MessageQueue queue = me.mQueue;

// Make sure the identity of this thread is that of the local process,
// and keep track of what that identity token actually is.
Binder.clearCallingIdentity();
final long ident = Binder.clearCallingIdentity();

for (;;) {
    Message msg = queue.next(); // might block
    if (msg == null) {
        // No message indicates that the message queue is quitting.
        return;
    }

    // This must be in a local variable, in case a UI event sets the logger
    Printer logging = me.mLogging;
    if (logging != null) {
        logging.println(">>>>> Dispatching to " + msg.target + " " +
                msg.callback + ": " + msg.what);
    }

    msg.target.dispatchMessage(msg);

    if (logging != null) {
        logging.println("<<<<< Finished to " + msg.target + " " + msg.callback);
    }

    // Make sure that during the course of dispatching the
    // identity of the thread wasn't corrupted.
    final long newIdent = Binder.clearCallingIdentity();
    if (ident != newIdent) {
        Log.wtf(TAG, "Thread identity changed from 0x"
                + Long.toHexString(ident) + " to 0x"
                + Long.toHexString(newIdent) + " while dispatching to "
                + msg.target.getClass().getName() + " "
                + msg.callback + " what=" + msg.what);
    }

    msg.recycleUnchecked();
}

    }

不断地调用的MessageQueue的next()方法,它的简单逻辑就是如果当前MessageQueue中存在mMessages(即待处理消息)，就将这个消息出队，然后让下一条消息成为mMessages，否则就进入一个阻塞状态，一直等到有新的消息入队。msg.target指的是Handler，然后用dispatchMessage方法

  1. public void dispatchMessage(Message msg) {
    if (msg.callback != null) { 
    handleCallback(msg);
    } else { 
    if (mCallback != null) {  
    if (mCallback.handleMessage(msg)) {  
        return;  
    }  
}  
handleMessage(msg);
    }
    }

最终调用handleMessage来实现功能。

4. Handler.post(Runnable r)方法

dispatchMessage中，若callback不为null，则handleCallback
private static void handleCallback(Message message) {
message.callback.run();
}

直接运行这个Runnable。不会新开线程，直接运行在主线程中.实现了把整段代码传进主线程中的功能。

5.loopPrepare()

在创建实例之前必须要有loopPrepare()方法。主线程里在初始化时已经调用

Oracle数据库安装中的坑

安装中遇到了几个坑，万一被人看到了呢

1.系统的默认共享一定要打开。不然会出现INS-30131错误。可以参考这个打开

先在控制面板的“服务”，看SERVER服务是否启动，如果没有启动，则将设置为自动或者手动，然后再选择启动。
开启系统的默认共享的方法
1.检查AutoShareServer和AutoShareWks注册表值是否为0。
2.找到注册表中的HKEY_LOCAL_MACHINE\System\CurrentControlSet\Services\LanmanServer\Parameters。
3.将下面子项中的AutoShareServer和AutoShareWks DWORD值改为1。
4.重启。通常Win2003、Win2000\XP会在启动时自动创建。
5.启动后，可以通过运行CMD命令进入命令行模式，再运行net share，再共享列表中会看到Admin$、C$\IPC$等默认共享。

2.解压文件两个要分开解压。。。不要懒。不然会遇到无法找到文件的错误

3.安装的最后几步有个用户管理，最好点进去看看默认的账户和密码。不然只能看这个修改用户名和密码等

Android笔记-Service

service的一些基本操作

service是系统四大组件之一，其作用是在后台静默的执行一些操作而不被使用者感知。

1.service的组成
onCreate(), onStartCommand, onDestroy. 其中，onCreate只在第一次创建时才会调用。
同时需要在manifest文件注册service。

2.如何启动一个service
与启动Activity类似，service的启动也是用intent，调用方法startservice(intent)即可。
也可以用putExtra()传递数据。

2.在service中实现功能
需要在service的onCreate或者onStartCommand()方法里创建一个Thread

4 .绑定service，实现活动和服务之间的通信
把Activity与Service绑定在一起，那就是Service要随着程序的退出而终止。在Activity中根据具体的场景来调用MyBinder中的任何public方法，即实现了Activity指挥Service干什么Service就去干什么的功能（调用class MyBinder extends Binder）

首先创建了一个ServiceConnection的匿名类，在里面重写了onServiceConnected()方法和onServiceDisconnected()方法，这两个方法分别会在Activity与Service建立关联和解除关联的时候调用。在onServiceConnected()方法中，我们又通过向下转型得到了MyBinder的实例，有了这个实例，Activity和Service之间的关系就变得非常紧密了
　　　　
首先通过使用Activity的bindService(Intent service,ServiceConnection conn,int flags)

bindService(new Intent(this,MyService.class),this,     Context.BIND_AUTO_CREATE);

然后编写类实现ServiceConnection 接口

   public void onServiceConnected(ComponentName name, IBinder service) {
    //System.out.println("Service connected");

}

@Override
public void onServiceDisconnected(ComponentName name) {
    //System.out.println("ServiceDisconnected");
}

同时重写service的onBind(Intent intent)方法

@Override
public IBinder onBind(Intent intent) {
    return new Binder();
}

5.IntentService
一般的服务启动后会一直运行，除非有stopSelf方法，和stopService方法才会停止，IntentService却可以自动结束。 但只需调用onHandleIntent() 方法即可实现服务自动结束。

6.service 和 Thread 的关系
没有任何关系。
service 用于Activity被销毁后，在后台继续执行动作。也就是说，如果在服务里执行耗时长的动作，还是会ANR。 而子线程就用于避免这种状况。
