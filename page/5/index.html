Android 笔记：activity

最近开始学Android，用的书是《第一行代码》，同时看极客学院的视频。

Android最基础的部分是activity，就是活动。Android上一切都建立在activity上。

创建活动

在as上创建很简单，只用选blank activity就把所有工作完成了。实际上创建一个activity要完成的工作有

  • 新建类。继承（extends）activity基本类
  • 创建和加载布局
  • 在Manifest文件里注册

Activity的生命周期

生命周期分为onCreate、onStart、onResume、onPause、onRestart、onStop、onDestroy。除onDestroy外前面的两两……………………………………………………………….配对。

启动模式

启动模式分为Standard（默认）、SingleTask、SingleTop、SingleInstanse.

1.Standard

standard模式就是无论什么情况下，一旦激活activity，就会在任务栈里新建一个实例，退出就会销毁。

2.SingleTop

这个模式是，如果激活自己，就不会创建。即当前activity若是在栈顶，就不会新建实例。其他情况还是会创建新的。

3.SingleTask

这个模式下的一个activity只会有一个实例，每次激活的都是这个实例。
只需要把此Activity放入栈顶，并把该Activity以上的Activity实例都pop这条待验证

4.SingleInstance

这个模式下激活会启用一个新的栈结构存放activity，与原结构完全独立。

SICP笔记1.1

全称《Structure and Interpretation of Computer Program》《计算机程序的构造和解释》。同时借助优酷上这个教程。

1.1主要介绍了Lisp语言的方言scheme的语法。scheme的语法可以说非常简单也非常奇特(相对于c++和java)。

  1. 运算符前置，语句用括号括起来。
    比如
    (* 1 2)
    (+ (+ 1 2) (* 2 5))
  2. define可以定义变量，也可以定义过程。
  3. 条件表达式cond，if。
  4. 过程作为黑箱的思想。类似函数和oop，把求解过程封装，只留接口。

这本书被很多人奉为神书，抱着试一试的心态看了一点。果然很“神”————不怎么看得懂。只有硬着头皮看了，所以速度不会太快，慢慢来吧。。

搭建过程

搭建过程主要参考了 hexo你的博客 和 Hexo搭建Github静态博客 两篇文章， 其中有些步骤略有不同。

环境准备

安装Node

Node.js官网下载相应平台的最新版本，按默认配置安装即可。

安装Git

Git。

安装GitHub

GitHub官方出品的软件。图形的界面用起来很傻瓜。免去了设置SSH密钥的繁琐。

建立blog

主要步骤分为配置GitHub和Hexo两个。

1.配置Hexo

1.1 Setup your blog

首先打开Git Bash，执行如下命令
‘ $ npm install -g hexo ‘
建立一个文件夹（比如E:/Hexo)，在此文件夹后右键，选Git Bash Here。
执行下面的命令
$ hexo init
然后执行
$ npm install
Hexo就配置到Hexo文件夹中了。

1.2 Start the server

运行下面的命令（在E:/Hexo下）
$ hexo server
表明Hexo Server已经启动了，在浏览器中打开 http://localhost:4000/
这时可以看到Hexo已为你生成了一篇blog。
你可以按Ctrl+C 停止Server。

1.3 Create a new post

新打开一个git bash命令行窗口，到Hexo目录下，执行下面的命令
$ Hexo new post "My New Post"
刷新 http://localhost:4000/
可以发现已生成了一篇新文章 “My New Post”。

4. Generate static files

执行下面的命令，将markdown文件生成静态网页。
$ Hexo generate
此时Public目录下会有blog的全部文件，然后开始下一步。

2.GitHub

首先注册GitHub账号。

打开GitHub软件，点左上角加号，直接在建立一个repository，名为【your_user_name.github.io】。选择一个本地目录。

把Hexo/public文件下的文件全部复制到repository的本地目录中，然后点GitHub软件右上角的Publish。
